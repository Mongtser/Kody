Car Bar

Mobility assist device | Senior design project

Mechanical Design

Designed a mobility-assist device intended to help users enter and exit a vehicle with greater stability, confidence, and ease. This project pushed me to balance user needs, structural loading, real-world fit, and practical prototyping.

  • Defined the design around support, accessibility, ease of storage, and compatibility with the vehicle environment.
  • Modeled the assembly and evaluated high-stress regions with stress and fatigue analysis to guide the design.
  • Worked through iteration tradeoffs between usability, manufacturability, and repeated loading durability.
  • Built a functional prototype to validate the concept in a realistic use case rather than stopping at CAD.
SolidWorks FEA Prototype Fabrication Human-Centered Design

Smart Gun Case

Embedded systems project

Hardware + Software

Built a multi-factor security system that combined biometric verification, keypad access, and display feedback into a single working prototype. The focus was reliability, integration, and a user flow that felt simple even with layered security.

  • Wrote more than 300 lines of Arduino code for access logic, input handling, and system response.
  • Integrated biometric scanning, keypad control, and OLED feedback into one coordinated prototype.
  • Debugged hardware-software interaction issues to improve consistency and responsiveness.
  • Validated the design through repeated test cycles instead of treating the build as a one-pass demo.

Process

How I approach engineering problems

I like work that starts with a real constraint and ends with something testable. Across school, projects, and internships, the pattern below shows up again and again.

Frame the problem Understand the user need, operational constraint, or failure mode before jumping into the design.
Build a workable concept Use CAD, rough calculations, or quick prototypes to get to something real fast enough to evaluate.
Test the weak points Look for where wear, stress, usability, or integration issues are most likely to show up.
Revise with evidence Use the result, not the ego, to decide what to change next.
