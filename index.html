Car Bar

Mobility assist device | Senior design project | Fall 2024 - Spring 2025

Mechanical Design

Led the mechanical design process for a mobility-assist device targeting vehicle entry and exit, from initial concept framing through iterative CAD, analysis, and prototype validation.

  • Translated user needs around support, accessibility, and real-world vehicle compatibility into design constraints that shaped every concept direction.
  • Used SolidWorks to model multiple assembly iterations, applying stress and fatigue analysis to identify weak points and justify geometry changes between revisions.
  • Made explicit tradeoffs between usability, manufacturability, and repeated-load durability, documenting the reasoning behind each revision rather than just the change itself.
  • Brought the design to a functional prototype stage, validating core concepts against realistic use conditions before the project concluded.

Smart Gun Case

Embedded systems project | Spring 2023

Hardware + Software

Built an embedded systems prototype focused on layered physical access control using biometric access, keypad input, and display feedback.

  • Wrote 300+ lines of Arduino code for access logic, keypad handling, fingerprint flow, and system response.
  • Integrated biometric scanning, keypad control, and OLED feedback into one prototype so the system could move through authentication and user feedback as one sequence.
  • Diagnosed and resolved timing conflicts between the fingerprint sensor and keypad input logic to ensure reliable sequential access control.
  • Ran repeated access-sequence tests across all input combinations to confirm the system responded correctly under normal use and edge-case inputs before calling it done.
The prototype enclosure was built within budget and time constraints, so the priority was proving the access-control logic, component integration, and user feedback flow in a working physical build rather than polishing the final packaging.

Process

How I approach engineering problems

Frame the problem Understand the user need, operational constraint, or failure mode before jumping into the design. See: Car Bar.
Build a workable concept Use CAD, rough calculations, or quick prototypes to get to something real fast enough to evaluate. See: Car Bar and Smart Gun Case.
Test the weak points Look for where wear, stress, usability, or integration issues are most likely to show up. See: Smart Gun Case and JBT.
Revise with evidence Use the result, not the ego, to decide what to change next. See: Car Bar, Smart Gun Case, and Excelsior.
